Project 2: Retail Customer SQL Analysis

Relational Database Design • SQL Analytics • Business Intelligence

📌 Project Overview

This project demonstrates the complete design, development, and analytical evaluation of a relational retail database using SQLite via DBBrowser. The objective was to simulate a real-world retail environment and apply SQL querying techniques to extract actionable business insights from transactional data.

Workflow:

  • Designing the ERD
  • Creating a normalised relational database
  • Implementing primary and foreign key constraints
  • Populating structured tables with transactional data
  • Writing SQL analytical queries
  • Interpreting analytical outputs for decision-making

💻 SQL Queries & Results

Total Revenue

Calculates total business revenue using SUM aggregation.

Top Customers

Identifies highest-spending customers using JOIN + GROUP BY.

Best Selling Products

Ranks products by total quantity sold.

Monthly Revenue Trend

Tracks revenue patterns using date functions.

Average Order Value

Subquery calculates average basket size.

High Value Orders

HAVING clause filters orders exceeding £500.

Revenue by City

Analyses geographic revenue contribution.

Revenue by Category

Evaluates performance by product category.

📊 Key Insights

Total generated revenue was £3360.00.
Keyboard was the best-selling product.
London customers contributed the largest revenue.
July showed consistent revenue across weeks.
Laptops generated £900 compared to phones at £600.
Average order value was £373.33.

📈 Business Recommendations

Focus marketing efforts on high-value electronics such as laptops.
Implement loyalty campaigns targeting London customers.
Bundle accessories with premium devices to increase basket size.
Monitor monthly sales trends for forecasting.
